Breakpoints view 에서는 workspace 에 설정된 breakpoint 들을 보여주고 관리 할 수 있습니다.
Breakpoints view 에는 Java 개발에서는 사용 가능하지만 javascript 에서는 사용할 수 없는 기능들이 다수 포함되어 있습니다. 이 문서에서는 javascript 에서 사용할 수 없는 기능들에 대해서 기술하지 않습니다.
 

 
1
선택 해제하면 breakpoint 가 동작하지 않게 됩니다.
 
2
Breakpoint 가 위치하는 파일과 라인 번호가 표시됩니다.
 
3
선택된 breakpoint 를 삭제합니다.
 
4
Breakpoint 를 모두 삭제합니다.
 
5
Breakpoint 가 설정된 소스 위치로 이동합니다.
 
6
선택하면 모든 breakpoint 가 동작하지 않습니다.
 
7
트리를 모두 펼칩니다.
 
8
트리를 모두 접습니다.
 
9
다음과 같은 추가 메뉴를 사용할 수 있습니다.
 

 
10: Layout > Vertical
선택된 breakpoint 에 대한 세부 설정을 할 수 있는 detail pane 을 하단에 배치합니다.
 
11: Layout > Horizontal
Detail pane 을 오른쪽에 배치합니다.
 
12: Layout > Automatic
Breakpoints view 의 크기 비율에 따라 detail pane 의 위치를 자동으로 조정합니다.
 
13: Layout > Breakpoints View Only
Detail pane 을 보이지 않습니다.
 

 
14: Group By > Breakpoints
Grouping 하지 않습니다.
 
15: Group By > Breakpoint Types
Breakpoint 를 type 에 따라 grouping 합니다.
 
16: Group By > Breakpoint Working Sets
Breakpoint 를 Breakpoint working set 에 따라 grouping 합니다.
 
17: Group By > Files
Breakpoint 를 File 에 따라 grouping 합니다.
 
18: Group By > Projects
Breakpoint 를 Project 에 따라 groupoing 합니다.
 
19: Group By > Resource Working Sets
Breakpoint 를 Resource working set 에 따라 grouping 합니다.
 
20: Group By > Advanced...
Breakpoint 를 여러 계층으로 grouping 할 수 있습니다.
 
21: Select Default Working Set...
Breakpoint 를 등록하면 추가된 breakpoint working set 을 지정합니다.
 
22: Deselect Default Working Set
현재 설정된 working set 을 선택 해제합니다.
 
23: Working Sets...
Breakpoint working set 을 추가/삭제/수정할 수 있습니다.
