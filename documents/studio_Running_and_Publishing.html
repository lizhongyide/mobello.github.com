Run Mobello Application

Project Explorer 에서 마우스 오른쪽 버튼으로 project 를 선택해서 컨텍스트 메뉴를 띄운 후 Run As > Run Mobello 를 선택하면 Mobello Emulator 에서 application 이 실행됩니다.
 

 
또는 toolbar 의 Run 버튼으로도 실행할 수 있습니다.
 

 
아니면 단축키 Ctrl + F11 을 사용해도 실행됩니다.
 

Debug Mobello Application

 
Project Explorer 에서 마우스 오른쪽 버튼으로 project 를 선택해서 컨텍스트 메뉴를 띄운 후 Debug As > Debug Mobello 를 선택하면 debug mode 로 application 이 실행됩니다.

또는 toolbar 의 Debug 버튼으로도 debug mode 로 실행할 수 있습니다.

아니면 단축키 F11 을 사용해도 debug mode 로 실행됩습니다.
 

Debug Configuration

 
다음과 같이 Toolbar 에서 Debug > Debug Configurations... 를 선택하면 Debug Configuration 다이얼로그가 나타납니다.
 

 
Debug Configuration 다이얼로그에서는 Exception Catch Mode 를 선택할 수 있습니다.
 

 
1: Uncaught Exception
Uncaught exception ==(try catch 로 잡히지 않는 exception)== 이 발생하면 debug 할 수 있도록 실행을 중단합니다.
 
2: All
모든 exception 발생시 debug 할 수 있도록 실행을 중단합니다.
 
3: None
Exception 이 발생해도 실행을 중단하지 않습니다.
 
4: Debug
선택된 Mobello Application 을 debug 모드로 실행합니다.
 
5
실행 가능한 mobello application 들을 보여줍니다.
 

Publishing

 
완성된 application 을 web server 에 deploy 하거나 native app 에 packaing 하기 위해서는 먼저 webapp 으로 publish 해야 합니다.
Publish 하기 위해서는 project explorer 에서 project 를 마우스 오른쪽 버튼으로 선택한 후 Publish as Webapp... 을 선택합니다.
 

 
Publish as Webapp... 을 선택하면 다음과 같은 다이얼로그가 나타납니다.
 

 
1: To directory
Publish 할 directory 의 경로를 지정합니다.
 
2: Overwrite existing files without warning.
경고 없이 publish 할 파일을 덮어 씁니다.
 
3: Clear destionation directory before publishing.
Publish 하기 전에 directory 안의 파일을 모두 삭제합니다.
