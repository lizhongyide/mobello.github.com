Objective-C 나 Android SDK 를 사용해서 native app 으로 만들지 않더라도 하이브리드 앱빌더를 Mobello 와 같이 사용하면 앱스토어나 안드로이드 마켓에 등록할 수 있는 webapp 을 만들 수 있습니다. 또한 하이브리드 앱 빌더를 활용하면 카메라나 주소록같은 device  resource 를 사용하는 app 도 Mobello 를 기반으로 만들 수 있습니다. 이 문서는 대표적인 하이브리드 앱빌더인 Appspresso 를 Mobello 와 함께 사용해서 mobile hybrid app 을 만드는 과정을 설명하고 있습니다.

Install Appspresso

다음 update site 를 통하여 기존에 설치된 Mobello Studio 에 Appspresso 를 추가로 설치합니다.

  • http://appspresso.com/updates/1.0/

자세한 설치 방법은 다음 문서에 설명되어 있습니다.

  • Install Appspresso 1.1.2 to existing Mobello Studio 1.1

개발된 app 을 android package 파일(apk) 으로 packaging 하거나 PC 에서 emulator 로 테스트하기 위해서는 Android SDK 를 설치해야 합니다. Android SDK 설치 방법은 다음 문서에서 자세히 설명되어 있습니다.

  • Install Android SDK & Create AVD

Create Mobello with Appspresso Project

모든 설치가 완료 되었으면 Mobello with Appspresso Project wizard 를 사용해서 project 를 생성합니다.

Project name 과 version, vendor 를 입력하고 Finish 를 선택합니다.

생성된 project 의 main.scene 파일에 다음과 같이 Button 컴포넌트를 하나 배치합니다.

Attributes view 의 Events 에서 onTap 에 event handler 를 지정하고 main.scene 을 저장합니다.

이제 main.js 를 열고 방금 Button 컴포넌트에 지정했던 event hanlder 를 생성합니다. Event hanlder 에서는 현재 active 되어 있는 appspresso feature 들을 alert 으로 띄우도록 구현했습니다.

main.js
handletap: function() {
	var features = deviceapis.listActivatedFeatures(); 
	for (var i=0; i < features.length; i++) { 
	  alert("The Feature " + features[i].uri + " has been activated"); 
	}
},

Run in Android Emualtor

만들어진 app 을 Android Emulator 에서 실행해보도록 하겠습니다. 마우스 오른쪽 버튼으로 project 를 선택하고 Run As > Android 장치에서 Appspresso 애플리케이션 실행을 선택합니다.

장치 선택 화면에서 실행하려는 android virtual device 를 선택하고 OK 를 선택합니다.

잠시 후 에뮬레이터가 실행 되면서 다음과 같이 app 이 실행된 것을 볼 수 있습니다.

APK 파일 생성

Project Explorer 뷰에서 project 를 오른쪽 마우스버튼으로 클릭한 후 컨텍스트 메뉴에서 Export 메뉴를 선택합니다. 
Export 다이얼로그에서 Appspresso > Android 애플리케이션 (.apk 파일)을 선택합니다.

앱을 Android 마켓에 업로드하려면 반드시 서명을 해야 합니다. 앱 서명에 대한 자세한 사항은 Signing Your Applications 을 참조하시기 바랍니다.
키스토어 파일 경로 항목의 찾아보기 버튼을 클릭하고 키스토어 파일의 위치를 지정한 후 키스토어 비밀번호를 입력합니다.
키 별명 목록에서 별명을 선택하고 키 비밀번호를 입력합니다.
apk 파일이 저장될 경로를 입력하고 Finish 버튼을 클릭하면 지정된 경로에 apk 파일이 생성됩니다.
