Eclipse 구성요소
Workbench : eclipse의 사용자 인터페이스 입니다. 윈도우의 집합이며, 각 윈도우는 위 화면과 같은 구성요소들로 이루어져 있습니다.
Perspective bar : 스튜디오의 맨 위, 오른쪽에 위치하고 있는 toobar 입니다. Perspective bar 를 이용하여 perspective 간의 이동을 쉽게 할 수 있습니다.
View : 정보를 표시하는데 사용됩니다. 하나의 workbench 내에는 같은 view가 2개 이상 존재할 수 없습니다.
Perspective :각종 editor 와 view 의 모임. 작업의 종류에 따라 다른 모양이 됩니다. 
예를 들어, 'Design'perspective 선택하면 Mobello Application 을 만들기 위한 Visual Designer, QuickStyler, Attributes view 등으로 바뀌고 'debug'를 선택하면 Mobello Application 을 디버깅하기 위한 view들로 바뀝니다. 
플러그 인의 설치에 따라 플러그 인과 관련된 퍼스펙티브를 선택할 수 있습니다.

Design Perspective란?

Mobello Project 개발을 위한 도구를 제공합니다. Design Perspective 의 구성은 다음과 같습니다.
 

Design Perspective의 구성요소

  1. Toolbar - Toobar 에서는 새로운 project 생성, 파일의 저장과 인쇄, Zoom in/out 및 기본 디자인 사이즈 설정과 같은 visua designer 에 필요한 기능들이 나열되어 있습니다.
  2. Project Explorer - Project 자원(프로젝트, 폴더, 파일)의 계층구조를 출력합니다. 파일의 저장, 수정, 삭제가 가능합니다.
  3. Outline view - 선택 된 designer 에 포함된 component 들의 구조적인 개요를 나타냅니다. 컴포넌트들의 배치 순서와 포함 관계를 파악할 수 있습니다.
  4. Palette view - 디자인 가능한 component 리스트를 나타내는 View. 이 Palette view 를 이용해 D&D 방식으로 손쉽게 디자인 할 수 있습니다.
  5. Visual Designer - Design perspective 의 주요 editor 로 palette 를 이용하여 mobello project 디자인을 할 수 있습니다.
  6. QuickStyler - 빠른 스타일 입력을 위한 View. 폰트 속성이나 색상, 위치 관련 정보를 수정할 수 있습니다.
  7. Attributes - Component 상세 정보 변경을 위한 View. Component 자체 속성 및 CSS, Event 정보를 수정할 수 있습니다.
