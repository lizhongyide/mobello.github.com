Class tau.rt.Runtime

  • HIERARCHY
    tau.TObject
   |
   +--tau.rt.EventDelegator
      |
      +--tau.rt.Runtime
  • Defined in:
    tau.rt.js

모듈(tau.rt.Module)의 동작을 관리한다.

내부적으로 사용되는 시스템 이벤트는 다음과 같다.

  • tau.rt.Event.RT_INSTALL: 모듈을 Runtime에 등록한다.
  • tau.rt.Event.RT_UNINSTALL: 모듈을 Runtime으로부터 제거한다.
  • tau.rt.Event.RT_START: 모듈을 실행시킨다.
  • tau.rt.Event.RT_STOP: 모듈의 동작을 중지시킨다.
  • tau.rt.Event.RT_LAUNCH: Runtime을 구동시킨다.
  • tau.rt.Event.RT_TERMINATE: Runtime을 중지시킨다.

  • install(apps)
    명시된 모듈 configuration정보를 Runtime에 등록한다.

    Module이름이 명시되면 해당 모듈의 configuration정보를 지정된 패스로 부터(appname/config.json) 로딩한다. 만약 이름이 명시되지 않으면 Dashboard를 위한 configuration정보를 로딩한다.

    명시된 앱이 Runtime에 등록되기 위해서는 반드시 configuration정보가 필요하며 configuration정보의 포맷은 다음과 같이 JSON형태로 기술한다.

    명시된 모듈의 설치가 완료되면 tau.rt.Event.RT_INSTALLED 이벤트가 발생된다. 이때 payload객체는 설치된 앱들의 configuration정보가 된다. 

    // config.json
config({
  name: 'appname',
  version: '1.0.0', 
  classname: 'foo.Bar'
  ...
});
    Parameters:
    {Object | Array} apps
    install할 모듈의 정보

    브라우저의 URL로 부터 Mobello의 Runtime을 launch시킨다.

    URL 포맷은 다음과 같다. 

    http://<domain.com>/launcher.html?app=<appname>

  • start(name)
    명시된 이름의 모듈의 Lifecycle을 시작한다.

    명시된 이름에 해당하는 Module이 정상적으로 설치되어 있다면 새로운 객체를 생성하고 실행시킨다. 그렇지 않으면 지금까지 실행된 Module들중 가장 최근에 실행한 Module을 구동시킨다. 마지막으로 현재 active모듈이 존재하지 않으면 Dashboard를 구동시킨다.

  • stop(name)
    명시된 이름에 해당하는 Module의 Lifecycle을 중지하고 destroy시킨다.

    Module이름이 명시되고 해당 Module이 존재하면 그 모듈을 중지시키고 그렇지 않으면 실행된 Module들 중 가장 최근에 실행된 Module을 찾아 중지 시킨다.

    Module이 중지되면 Control을 그 다음으로 최근에 실행된 Module로 넘긴다음 lifecycle을 실행시킨다.

    Parameters:
    {String} name
    Module 이름/id, undefined 이면 현재 실행중인 Module을 중지시킨다.
