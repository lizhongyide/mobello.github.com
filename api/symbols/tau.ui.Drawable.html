Class tau.ui.Drawable

  • HIERARCHY
    tau.TObject
   |
   +--tau.ui.Drawable
  • Defined in:
    tau.ui.comp.js

Drawable은 UI 컴포넌트 객체들을 랜더링하기 위해 사용되는 DOM element를 캡슐화하는 클래스이다. Drawable은 독립적으로 인스턴스를 생성하지 않고 다른 컴포넌트 클래스에 Mixin 되어 사용된다.

#draw 함수를 호출하면 객체가 관리하는 DOM element를 부모 element의 하위 노드로 추가하는 것이다.
이전에 랜더링한 컨텐트를 클리어하기 위해 #clear함수를 호출해서 부모 DOM element로 부터 관리하고 있는 DOM element를 삭제할 수 있다.
나머지 멤버 함수들은 객체가 관리하고 있는 DOM element를 조작하는데 도움을 준다. 특히 #getDOM함수는 컴포넌트의 UI를 직접 조작하기 위해 클라이트에서 사용하는 DOM 접근자이다.

  • {Boolean} draw(parent, refresh, refChild)
    인스턴스를 draw한다.

    인스턴스의 최상위 DOM element를 부모가 되는 DOM element에 추가하고, 기본 스타일 클래스, 스타일 클래스 설정된 값을 최상위 DOM element에 추가한다.

    이 함수를 오버라이딩하려면 부모 DOM element가 그것의 하위 DOM element들이 그려지기 전에 DOM 트리에 추가되는 것을 보장해야한다. 

      
  draw: function (parent, refresh) {
    ...
    // calling draw will ensure parent & this.getDOM() are attached
    if (tau.ui.Drawable.prototype.draw.apply(this, arguments)) {
      for (var i = 0; i < children.length; i++) {
        children[i].draw(this.getDOM(), refresh); 
      }
      return true;
    }
    return false;
  }

    -랜더링되는 순서

    1. #beforeRender: 전처리를 수행한다.
    2. 최상위 DOM element를 parent DOM element에 추가한다.
    3. tau.ui.Drawable.id를 DOM element에 설정하고
    4. 기본 스타일 클래스와, 그 외 스타일 클래스를 DOM element에 설정한다.
    5. #render : 추가적으로 최상위 DOM element 하위에 설정되어야 하는 속성이나, 그려져야 하는 부분에 대해 처리한다.
    6. #afterRender : 랜더링이 완료된 후에 계산되어야 할 작업들이 있는 경우 후처리를 수행한다.

    주의 : 사용자가 직접 draw 메소드를 호출하지는 않는다.

    최상위 DOM element의 이전 노드에 해당하는 DOM element
